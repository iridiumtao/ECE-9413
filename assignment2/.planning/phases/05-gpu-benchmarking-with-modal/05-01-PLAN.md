**Local entrypoint (per D-09):**
```python
@app.local_entrypoint()
def main(
    tests_only: bool = False,
    bench_only: bool = False,
    gpu: str = "T4",
    bits: int = 32,
):
    gpu_upper = gpu.upper()
    if gpu_upper not in _GPU_FUNCTIONS:
        raise ValueError(f"Unknown GPU: {gpu!r}. Choose from: {sorted(_GPU_FUNCTIONS)}")
    _GPU_FUNCTIONS[gpu_upper].remote(
        tests=not bench_only,
        bench=not tests_only,
        bits=bits,
    )
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local → Modal cloud | `add_local_dir(".")` uploads the entire working tree to Modal's infrastructure |
| Modal container → PyPI | `pip_install` fetches packages from PyPI at image build time |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Information Disclosure | `add_local_dir` ignore list | mitigate | Exclude `.env`, `.env*`, `*.env.*`, `*.key`, `*.pem`, `*.secret`, `*.crt` in the ignore list passed to `add_local_dir` — prevents accidental upload of credential files present in the working directory |
| T-05-02 | Information Disclosure | Modal container stdout | accept | Benchmark output is printed to stdout — no secrets are computed or printed by `tests.benchmark`; risk is negligible |
| T-05-03 | Tampering | `git checkout <hash> -- student.py` workflow | accept | Only `student.py` is checked out per hash; all other project files stay at HEAD; no integrity risk beyond the expected diff |
| T-05-04 | Elevation of Privilege | PyPI supply chain | accept | Packages are well-known (`jax`, `numpy`, `pytest`, `sympy`, `rich`); version pinning is out of scope for a coursework project |
</threat_model>

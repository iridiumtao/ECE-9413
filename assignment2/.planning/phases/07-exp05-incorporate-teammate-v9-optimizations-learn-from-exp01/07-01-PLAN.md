3. Add `_sumcheck_32_exp05` after the Barrett helpers. This is the CPU kernel. Key differences from current sumcheck_32:

   a. Deferred mod reduction: instead of `% q64` after each term and then accumulating, accumulate all terms in a raw uint64 accumulator and apply ONE `% q64` at the end per t-point. Safety bound: each term_product element is < q < 2^32; with up to N=2^20 elements and up to ~10 terms, the raw sum is < 10 * 2^20 * 2^32 = 2^55.3 < 2^64. So no overflow.

   b. No-mul shortcuts preserved for t=2 and t=3: use the existing inline uint64 expressions `(2*o64 + q64 - z64) % q64` and `(3*o64 + 2*q64 - 2*z64) % q64` — but the % q64 here is per-element (array op), not a scalar reduction. This is correct.

   c. Running-add for t>=4 only: compute `diffs` once before the t loop; for each t>=4 iteration, `running = {var: mod_add_32(running[var], diffs[var], q32) for var in tables}`. Initialize running at t=3 values (the no-mul shortcut output).

   d. The fold step uses `mle_update_32(evens[var], odds[var], r, q=q)` as before.

   e. claim0 is derived from `mod_add_32(round_evals_list[0][0], round_evals_list[0][1], q)` — unchanged.

   The full implementation of `_sumcheck_32_exp05` uses the **advance-before-use** pattern for t>=4. The loop structure is:

   ```python
   @functools.partial(jax.jit, static_argnames=("expression", "num_rounds"))
   def _sumcheck_32_exp05(eval_tables, *, q, expression, challenges, num_rounds):
       """Exp05 CPU kernel: Exp03 no-mul shortcuts + v9 deferred mod + running-add for t>=4."""
       degree = max(len(term) for term in expression)
       used_vars = set().union(*expression)
       tables = {v: jnp.asarray(eval_tables[v], dtype=jnp.uint32) for v in used_vars}
       q32 = jnp.asarray(q, dtype=jnp.uint32)
       q64 = jnp.asarray(q, dtype=jnp.uint64)

       round_evals_list = []

       for i in range(num_rounds):
           evens = {var: tbl[::2]  for var, tbl in tables.items()}
           odds  = {var: tbl[1::2] for var, tbl in tables.items()}

           if degree >= 4:
               diffs = {var: mod_sub_32(odds[var], evens[var], q32) for var in tables}
           running = {}  # initialized at end of v=3 iteration; used from v=4 onward

           g_i_evals = []

           for v in range(degree + 1):
               # ADVANCE-BEFORE-USE: advance running BEFORE computing term_product for v>=4.
               # running starts as the t=3 per-variable values (seeded at end of v=3 iter).
               # For v=4: running becomes t=3+diffs = t=4 values. For v=5: t=5. Etc.
               if v >= 4 and degree >= 4:
                   running = {var: mod_add_32(running[var], diffs[var], q32) for var in tables}

               # Accumulate all terms in raw uint64; apply single % q at end (deferred mod).
               g_i_raw = jnp.uint64(0)

               for term in expression:
                   if v == 0:
                       term_product = evens[term[0]]
                       for var in term[1:]:
                           term_product = mod_mul_32(term_product, evens[var], q)
                   elif v == 1:
                       term_product = odds[term[0]]
                       for var in term[1:]:
                           term_product = mod_mul_32(term_product, odds[var], q)
                   elif v == 2:
                       z64 = evens[term[0]].astype(jnp.uint64)
                       o64 = odds[term[0]].astype(jnp.uint64)
                       term_product = ((2*o64 + q64 - z64) % q64).astype(jnp.uint32)
                       for var in term[1:]:
                           z64_v = evens[var].astype(jnp.uint64)
                           o64_v = odds[var].astype(jnp.uint64)
                           factor = ((2*o64_v + q64 - z64_v) % q64).astype(jnp.uint32)
                           term_product = mod_mul_32(term_product, factor, q)
                   elif v == 3:
                       z64 = evens[term[0]].astype(jnp.uint64)
                       o64 = odds[term[0]].astype(jnp.uint64)
                       term_product = ((3*o64 + 2*q64 - 2*z64) % q64).astype(jnp.uint32)
                       for var in term[1:]:
                           z64_v = evens[var].astype(jnp.uint64)
                           o64_v = odds[var].astype(jnp.uint64)
                           factor = ((3*o64_v + 2*q64 - 2*z64_v) % q64).astype(jnp.uint32)
                           term_product = mod_mul_32(term_product, factor, q)
                   else:
                       # v>=4: running already advanced to the t=v value above.
                       term_product = running[term[0]]
                       for var in term[1:]:
                           term_product = mod_mul_32(term_product, running[var], q)

                   g_i_raw = g_i_raw + jnp.sum(term_product.astype(jnp.uint64))

               g_i_evals.append((g_i_raw % q64).astype(jnp.uint32))

               # SEED running at end of v=3 so it is ready for the advance at v=4.
               if v == 3 and degree >= 4:
                   running = {}
                   for var in tables:
                       z64_v = evens[var].astype(jnp.uint64)
                       o64_v = odds[var].astype(jnp.uint64)
                       running[var] = ((3*o64_v + 2*q64 - 2*z64_v) % q64).astype(jnp.uint32)

           round_evals_list.append(jnp.stack(g_i_evals))

           if i < len(challenges):
               r = challenges[i]
               tables = {
                   var: mle_update_32(evens[var], odds[var], r, q=q)
                   for var in tables
               }

       round_evals = jnp.stack(round_evals_list)
       claim0 = mod_add_32(round_evals_list[0][0], round_evals_list[0][1], q)
       return claim0, round_evals
   ```

4. Add `_sumcheck_32_barrett_exp05` after `_sumcheck_32_exp05`. Same structure but using Barrett ops:
   - Replace `mod_sub_32(odds[var], evens[var], q32)` with `_mod_sub_branchless_u32(odds[var], evens[var], q_u64)`
   - Replace `mod_add_32(running[var], diffs[var], q32)` with `_mod_add_branchless_u32(running[var], diffs[var], q_u64)`
   - Replace `mod_mul_32(a, b, q)` with `_mod_mul_barrett_u32(a, b, q_u64, mu_u64)`
   - The no-mul shortcuts for t=2/t=3 can remain as inline uint64 (they already avoid modular multiply) — the element-wise `% q64` in the no-mul formula is not a scalar IDIV on GPU; it is an element-wise operation that benefits from Barrett differently. However, for GPU simplicity, leave the no-mul formulas unchanged (their element-wise `% q64` is the only place Barrett would help in them; this can be left as XLA % for now since the inline expression is already a single reduction).
   - The deferred scalar reduction at end of each t-point: replace `(g_i_raw % q64).astype(jnp.uint32)` with `_barrett_reduce_u64_to_u32(g_i_raw, q_u64, mu_u64)`
   - The fold step: replace `mle_update_32(evens[var], odds[var], r, q=q)` with `_mle_update_barrett_u32(evens[var], odds[var], r, q_u64, mu_u64)`
   - claim0: replace `mod_add_32(...)` with `_mod_add_branchless_u32(..., q_u64)`
   - Precompute `q_u64 = jnp.uint64(int(q))` and `mu_u64 = jnp.uint64(_barrett_mu_64(int(q)))` at the top of the function

       """Route CPU to _sumcheck_32_exp05, GPU to _sumcheck_32_barrett_exp05."""
       try:
           backend = jax.devices()[0].platform

               eval_tables, q=q, expression=expression,
               challenges=challenges, num_rounds=num_rounds,
           )

       )
